# 搅拌站后料场上料系统仿真软件 — PPT概要

---

## 第1页：项目概述

**标题：搅拌站后料场上料系统仿真软件**

**一、项目背景**
- 搅拌站后料场上料系统涉及多条皮带、中转斗、分料小车、配料站及高位储料仓，物理链路长、控制逻辑复杂
- 实际生产环境中，调度策略、清空逻辑、故障诊断等核心算法难以在真实设备上反复验证
- 需要一套高保真的仿真平台，用于控制策略开发、调度算法验证及操作培训

**二、系统定位**
- 基于PyQt5的工业级桌面仿真软件
- 完整模拟8条上料路线、19条皮带、7个中转斗、28+12个料仓、4个分料小车的物理行为
- 集成自动调度优化、故障诊断、下位机通信等完整功能链
- 支持独立部署的前后端解耦架构

**三、技术栈**
- Python 3.10+ / PyQt5 / NumPy / Pandas
- TCP JSON协议 / UDP二进制帧 / 多线程并发

---

## 第2页：核心功能

**一、物理仿真引擎**
- **8条上料路线 × 6状态状态机**（IDLE→MOVING_TO_TARGET→FEEDING→CLEARING→WAITING→STANDBY）
- **物料全链路仿真**：上料点→皮带运输→中转斗转运/囤积→分料小车定位→目标料仓投放
- **3种清空策略**：顺序策略(sequential)、反序策略(reverse)、换列策略(column_switch)，适配不同物理布局
- **实时料仓消耗**与中转斗0.195t/s速率释放

**二、自动调度系统**
- 独立TCP调度服务进程（端口8891~8894），遗传算法+贪心启发式优化
- 三级触发机制：紧急(<10%)、空闲(<70t)、预请求(≥80%)，120秒冷却
- 调度结果自动缓存，支持多轮序列执行和节能待机(STANDBY)

**三、故障诊断**
- 接近开关卡高/卡低、中转斗开关卡死/称重异常、皮带速度异常、小车传感器故障
- 本地诊断引擎+TCP远程诊断双模式，30秒去重窗口

**四、工业通信**
- TCP JSON下行（端口8888，0.3s周期）→ 下位机控制器
- UDP 57字节二进制帧（端口8889，CRC16校验）→ 实时控制总线
- 全部传感器、中转斗、皮带速度、小车位置数据实时推送

**五、可视化**
- 暗色工业风格2D实时动画
- 皮带运行状态、物料粒子、中转斗填充、分料小车平滑移动
- FEEDING状态闪烁箭头、料仓颜色分级（红→橙→黄→绿）
- 调度序列/状态监控/运行日志三栏信息展示

## 第3页：系统架构与解耦设计

**一、整体架构**
```
PyQt5 UI ←→ SimulationController ←→ [调度服务] [诊断服务] [下位机通信]
```
- 控制面板、仿真画布、状态监控三大UI模块
- SimulationController作为核心中枢，协调所有子系统

**二、解耦方案（decoupled_system）**
```
多前端(TCP) ←→ 后端服务(headless) ←→ [调度] [诊断] [下位机]
```
- 后端独立进程，零Qt依赖，纯Python标准库+socket
- TCP JSON行分隔协议，100ms快照推送
- 支持多个上位机同时接入同一后端

**三、设计亮点**
- **前后端分离**：仿真引擎与UI解耦，可独立运行、独立测试
- **开放协议**：JSON文本协议，任何语言/平台均可接入
- **多客户端**：支持监控、调度、诊断等多个上位机同时连接
- **可扩展**：新增路线/皮带/料仓仅需修改配置文件

**四、项目规模**
- 核心代码约60个Python文件，总计约15,000行
- 涵盖仿真引擎、调度优化、故障诊断、通信协议、UI渲染五大子系统
