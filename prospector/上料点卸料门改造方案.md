# 上料点卸料门改造方案

## 物理模型

```
上料点          卸料门         卸料斗            固定原料
──────────────────────────────────────────────────
feed2_1    →   gate_2_1   →   hopper_2_1    →   stone_powder
feed2_2    →   gate_2_2   →   hopper_2_2    →   aggregate_10mm / aggregate_20mm
feed3      →   gate_3_1   →   hopper_3_1    →   stone_powder / aggregate_10mm
           →   gate_3_2   →   hopper_3_2    →   同 hopper_3_1（必须同种原料）
```

- **feed2_1 / feed2_2**: 1 门 = 1 斗，门开 = 上料点出料，门关 = 停止
- **feed3**: 2 门共享 2 斗，任一门开 = 上料点出料；两个斗只能缓存同一种原料
- **FM 选 gate 规则**: 优先开有料的那扇门，都有料优先开 gate_3_1

## 通信协议变更

### FM ← 上位机（新增字段）

| 字段              | 格式                                                              | 用途                     |
|-------------------|-------------------------------------------------------------------|--------------------------|
| `feed_gate_states`  | `{"gate_2_1": false, "gate_2_2": false, "gate_3_1": false, "gate_3_2": false}` | 卸料门实际开关（故障诊断） |
| `feed_hopper_states`| `{"hopper_2_1": {"material": "stone_powder", "has_material": true}, ...}`       | 卸料斗原料类型和有无料     |

> 保留 `feed_material_states` 兼容，内部映射到 hopper 状态。

### FM → 上位机（命令替换）

| 旧命令                                                     | 新命令                                                                  |
|------------------------------------------------------------|-------------------------------------------------------------------------|
| `{"device":"feed_point","id":"feed2_1","action":"start"}`  | `{"device":"feed_gate","id":"gate_2_1","action":"open","material":"stone_powder"}` |
| `{"device":"feed_point","id":"feed2_1","action":"stop"}`   | `{"device":"feed_gate","id":"gate_2_1","action":"close"}`               |
| `{"device":"feed_point","id":"feed3","action":"start"}`    | `{"device":"feed_gate","id":"gate_3_1","action":"open","material":"stone_powder"}` |
| `{"device":"feed_point","id":"feed3","action":"stop"}`     | `{"device":"feed_gate","id":"gate_3_1","action":"close"}` + `{"device":"feed_gate","id":"gate_3_2","action":"close"}` |

> `silo_gate` / `silo_out` 不受影响。

## Gate 选择逻辑（feed3 关键）

```python
def _select_feed_gate(self, fp: str, material: str) -> str:
    """根据上料点和原料类型，返回应打开的卸料门 ID"""
    if fp == 'feed3':
        h1 = self._feed_hopper_states.get('hopper_3_1', {})
        h2 = self._feed_hopper_states.get('hopper_3_2', {})
        h1_has = h1.get('material') == material and h1.get('has_material')
        h2_has = h2.get('material') == material and h2.get('has_material')
        if h1_has:
            return 'gate_3_1'  # 优先 3-1
        if h2_has:
            return 'gate_3_2'
        return 'gate_3_1'  # 都没料也开 3-1（默认）
    # feed2_1 → gate_2_1, feed2_2 → gate_2_2
    return f"gate_{fp.replace('feed', '')}"
```

## 故障诊断逻辑

```
对于每个 feed_gate (gate_2_1, gate_2_2, gate_3_1, gate_3_2):
    如果该 gate 对应的上料点正在 FEEDING 且该 gate 是 FM 选中打开的那个：
        期望 = 开
    否则：
        期望 = 关
    如果实际 != 期望 → 报故障
```

## 注意事项

1. **silo_out 不变** — 高位仓卸料门 `silo_gate_S1-S12` 不受影响
2. **feed_point 命令全部替换** — 不留兼容，统一用 `feed_gate`
3. **真实上位机需同步新增** — `feed_gate_states` 和 `feed_hopper_states` 字段
4. **feed3 停止时需关闭两个 gate** — `close gate_3_1` + `close gate_3_2`
5. **delta 发送** — `_send_real_delta` 中 `feed_point` 的 off 映射改为 `feed_gate`
