# CLAUDE.md — 传感器模拟系统

## 项目概述

PyQt5 桌面应用，用于工业传感器数据模拟、控制策略生成和故障诊断。中文项目，中文用户交流。

## 工作流原则

### 写代码之前：先设计，后实现

1. **澄清需求** — 有疑问先提问，不要猜测
2. **设计方案** — 复杂功能先说方案，获得确认后再写代码
3. **每次一个任务** — 完成一个再开始下一个
4. **频繁提交** — 每个小任务完成后 commit

纯修 bug 时跳过设计环节，直接进入系统化调试。

### TDD：先写测试，再看它失败，最后写代码

```
没有失败的测试，就不写生产代码。
```

- 红：写一个失败的测试
- 绿：写最少代码让它通过
- 重构：绿灯状态下清理代码

即使没有正式测试框架（本项目当前无测试），复杂逻辑也要写临时验证脚本确认行为正确。

### 系统化调试：找到根因再修

```
不做根因调查，不许提修复方案。
```

1. 仔细读错误信息，逐字理解
2. 稳定复现问题
3. `git diff HEAD~5` 检查近期变更
4. 添加临时日志追踪数据流：
   ```python
   import sys
   print(f"DEBUG: x={x}", file=sys.stderr)
   ```
5. 提出单一假设，最小化验证
6. 同一 bug 修了 2 次还失败 → 停下来质疑架构，不要盲目继续试

不要"先改改看行不行"，也不要"先临时修一下以后再排查"。

### 完成后验证：有证据才能说"做完了"

```
没有新鲜的验证证据，不许宣称完成。
```

- 测试通过 → 展示本次运行的输出，不要凭记忆
- Bug 修复 → 验证原始症状已消失
- 功能完成 → 实际运行确认可用
- 不允许使用"应该"、"大概"、"似乎"来描述结果
- 提交前必须验证

## 编码规范

- 不写多余注释（代码自解释）
- 遵循 YAGNI：不添加当前不需要的功能
- 代码内禁止出现 TODO / FIXME / 占位符
- 修改现有文件，避免创建不必要的中间文件
- 不要引入过度抽象，三个相似行好过一个过早抽象

## 中文项目规范

### Commit Message

```
<type>(<scope>): <中文描述>

- <详细说明>
- <关联信息>
```

type 用英文：feat / fix / docs / style / refactor / perf / test / chore
scope 和 description 用中文，description 不超过 50 字，不加句号。

示例：
```
feat(数据管理): 添加传感器数据批量导出功能
fix(称重): 修复料斗重量计算溢出问题
```

### 文档排版

- 中英文之间加空格：`使用 Git 管理版本`
- 中文与数字之间加空格：`共 3 个传感器`
- 中文使用全角标点：`注意：需要先配置。`
- 技术术语保留英文：API、JSON、PyQt5、TCP

## 技术栈

- PyQt5 >= 5.15.0
- Python 3.x
- 架构：MVC（models/views/controllers）
- 通信：TCP socket（传感器数据接收）
