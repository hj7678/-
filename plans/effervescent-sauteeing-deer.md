# 下位机控制指令生成方案

## Context

仿真系统当前通过 `ControlStrategyGenerator` 生成皮带/中转斗/上料点等部件的目标状态，供前端动画和传感器数据生成使用。用户需要根据仿真系统在调度和上料各阶段的控制逻辑，生成对应的下位机硬件控制指令，包括 6 种指令类型：

1. **开/关上料点** — 控制上料点的启停
2. **启/停皮带** — 控制各皮带的启动/停止
3. **开/关中转斗** — 控制中转斗的开闭
4. **高位料仓出料口选择** — 选择出料的 S 仓编号
5. **小车目标位置** — 下发小车需要移动到的目标行号（1~7）
6. **小车左右分料拍板** — 控制小车翻板打向左或右

本方案枚举所有路线生命周期阶段和场景，定义每种场景下应生成的 6 类下位机指令。

---

## 路线状态总览

状态机：`IDLE → MOVING_TO_TARGET → FEEDING → CLEARING → WAITING → (FEEDING | IDLE)`

| 状态 | 含义 |
|------|------|
| IDLE | 路线空闲，未启用 |
| MOVING_TO_TARGET | 小车向目标行移动中 |
| FEEDING | 正常补料（状态一） |
| CLEARING | 清空余料（状态二） |
| WAITING | 停机待料（状态三） |

---

## 指令生成时机

下位机指令在状态转换的**进入时刻**生成并下发，持续运行的设备（皮带/上料点）在状态持续期间保持不变。指令生成与 `ControlStrategyGenerator` 并行执行，互不影响。

---

## 各状态详细指令

### 1. IDLE 状态

**场景**：路线未启用，或路线刚停止回到空闲。

**指令**：

| 指令类型 | 动作 | 适用条件 |
|----------|------|----------|
| 上料点 | 关闭 | 所有上料点 |
| 皮带 | 停止 | 该路线所有皮带 |
| 中转斗 | 关闭 | 该路线所有中转斗 |
| 料仓出料口 | 无动作 | — |
| 小车目标位置 | 无动作 | — |
| 小车分料拍板 | 保持当前 | — |

---

### 2. MOVING_TO_TARGET 状态

**进入时机**：用户启动路线（手动模式）或自动调度选中路线（自动模式），调用 `start_route()`。

**硬件行为**：
- 非终点皮带启动（物料运输路径准备就绪）
- 终点皮带停转（小车到位前不能送料）
- 中转斗保持关闭
- 上料点不启动
- 小车开始向目标位置移动

**指令下发顺序**：

```
步骤1: 启动非终点皮带（路线所有皮带中排除终点皮带）
步骤2: 确保终点皮带停止
步骤3: 下发小车目标位置 = ctx.cart_target_position
步骤4: 根据目标料仓列前缀下发小车分料拍板方向
步骤5: 确保所有中转斗关闭
步骤6: 确保上料点关闭
```

**小车分料拍板方向**：

| 小车 | 目标列 | left_divert | right_divert |
|------|--------|-------------|--------------|
| Cart1 (D7→P1) | P1 | True | False |
| Cart2 (D8→P2/P3) | P2 | True | False |
| Cart2 (D8→P2/P3) | P3 | False | True |
| Cart3 (D9→P4) | P4 | False | True |
| Cart4 (D6→S仓) | S1~S6 | True | True |
| Cart4 (D6→S仓) | S7~S12 | False | True |

---

### 3. FEEDING 状态

**进入时机**：小车到达目标位置后进入正常补料。

**硬件行为**：
- 所有皮带启动（包括终点皮带）
- 所有中转斗打开（物料直通）
- 上料点启动持续投料
- 储料仓路线选择料位最高的 S 仓出料

**指令下发顺序**：

```
步骤1: 按路线皮带顺序依次启动所有皮带（含终点皮带），相邻间隔 200ms
步骤2: 打开该路线所有中转斗
步骤3: 启动上料点
步骤4（仅储料仓路线）: 选择料位最高的 S 仓作为出料口
```

**上料点**：

| 路线 | 上料点 | 类型 |
|------|--------|------|
| route1 | feed1_1 | 标准 |
| route2 | feed1_2 | 标准 |
| route3 | feed2_1 | 标准 |
| route4 | feed2_2 | 标准 |
| route5 | feed2_2 | 标准 |
| route6 | feed3 | 标准 |
| route7 | feed3 | 标准 |
| route8 | silo_out | 储料仓出料 |
| route9 | silo_out | 储料仓出料 |

**储料仓路线 S 仓选择**：

| 路线 | 目标列 | 候选 S 仓 | 选择规则 |
|------|--------|-----------|----------|
| route8 | P4 | S1~S6 | 料位最高 |
| route9 | P2 | S7~S8 | 料位最高 |
| route9 | P3 | S9~S12 | 料位最高 |

---

### 4. CLEARING 状态 — 按策略区分

**进入时机**：FEEDING 期间目标料仓料位达到阈值。

三种策略的判定条件和阈值：

| 策略 | 判定条件 | 阈值 |
|------|----------|------|
| sequential（顺序） | 下次行号 < 本次行号 | 98% |
| reverse（反序） | 下次行号 > 本次行号，或无下次 | 95% |
| column_switch（换列） | 下次列前缀与本次不同 | 88% |

#### 4a. 顺序策略（sequential）— 同列从上往下

**场景**：如 P1-2 → P1-5。

**硬件行为**：关闭中转斗截留余料（斗内持续囤积、称重传感器持续增重），所有皮带保持运行直到余料全部清空。

**指令下发顺序**：

```
步骤1: 关闭所有中转斗
步骤2: 关闭上料点
步骤3: 等待路线所有皮带上物料清空 → 进入 WAITING
```

**例外 — 路线⑥和路线⑧（无中转斗）**：
这两条路线没有中转斗，顺序与反序完全相同：保持所有皮带开启，等待余料全部进入目标料仓即可。

#### 4b. 反序策略（reverse）— 同列从下往上或默认

**场景**：如 P1-5 → P1-2。

**硬件行为**：关闭中转斗截留余料，所有皮带保持运行直到物料清空。与顺序策略行为一致。

**指令下发顺序**：

```
步骤1: 关闭所有中转斗
步骤2: 关闭上料点
步骤3: 等待路线所有皮带上物料清空 → 进入 WAITING
```

#### 4c. 换列策略（column_switch）— 跨列换料

**场景**：D8 皮带上 P2↔P3 切换，仅关闭上料点，其余保持运转。

**指令下发顺序**：

```
步骤1: 关闭上料点
步骤2: 等待路线所有皮带上物料清空 → 进入 WAITING
步骤3: 进入 WAITING 后 → 停止终点皮带 → 关闭所有中转斗
```

---

### 5. WAITING 状态

**进入时机**：CLEARING 完成后（余料清空完毕）。

**硬件行为**：终点皮带停止、中转斗关闭、上料点关闭、小车维持当前位置。

**指令下发顺序**：

```
步骤1: 确保终点皮带停止
步骤2: 关闭所有中转斗
步骤3: 确保上料点关闭
```

**WAITING → FEEDING 恢复**：
- 若小车需移动到新位置 → 先进入 MOVING_TO_TARGET
- 若小车已在目标位置 → 直接进入 FEEDING

---

### 6. 停止路线（任意状态 → IDLE）

**指令下发顺序**：

```
步骤1: 停止路线所有皮带
步骤2: 关闭所有中转斗
步骤3: 关闭上料点
步骤4: 小车保持在当前位置
```

## 实现方式

### 新建文件：`controllers/machine_instruction_generator.py`

根据状态转换生成 6 类下位机指令数据结构：

```python
@dataclass
class MachineInstruction:
    instruction_type: str  # feed_point / conveyor / hopper / silo_outlet / cart_target / cart_divert
    device_id: str
    action: str            # start / stop / open / close / select / set_position / set_divert
    params: dict
    timestamp: float
```

### 调用点

在 `simulation_controller.py` 的状态转换方法中调用生成器获取指令列表，通过信号发送：

| 控制器方法 | 状态转换 | 生成指令 |
|------------|----------|----------|
| `start_route()` | IDLE→MOVING_TO_TARGET | 皮带启动 + 小车目标 + 分料方向 |
| `_immediate_cart_arrival()` / `_check_cart_arrival()` | MOVING_TO_TARGET→FEEDING | 全部皮带启动 + 斗开 + 上料点开 + S仓选择 |
| `trigger_clearing()` | FEEDING→CLEARING | 按策略：停皮带/关斗/关上料点 |
| `complete_clearing()` | CLEARING→WAITING | 停终点皮带 + 关斗 |
| `recover_feeding()` | WAITING→FEEDING | 全部皮带启动 + 斗开 + 上料点开 |
| `stop_route()` / `_complete_stop_route()` | 任意→IDLE | 全部停止 |

## 验证方式

1. **逐状态验证**：手动启动路线 → 观察每个状态转换时生成的指令列表是否正确
2. **策略切换**：配置不同调度序列触发三种清空策略 → 确认 CLEARING 进入指令差异
3. **储料仓路线**：启动 route8/route9 → 确认 S 仓选择指令按料位最高选出
4. **换列完整流程**：D8 调度 P2-3→P3-5 → 确认 column_switch 仅关闭上料点、斗保持开启
5. **停止恢复**：FEEDING 中停止 → 确认全部停止指令；WAITING 中恢复 → 确认重新启动指令
6. **指令输出**：在终端打印每次生成的指令列表，人工核对与仿真行为一致
