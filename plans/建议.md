● 工业部署差距分析

  🔴 致命级（必须解决）

  1. 无I/O抽象层

  当前直接读写仿真对象（small_bins[current_level]、conveyors.is_running）。真实PLC通过 Modbus/OPC UA 读写寄存器。缺少统一 I/O 接口：

  # 当前（耦合）
  level = self.small_bins[bin_id].current_level

  # 需要（抽象）
  level = self.io.read_tag(f"{bin_id}.level")  # 仿真→内存 / 真实→PLC

  2. 无PLC通信协议

  只有自定义TCP JSON，工业PLC不支持。需要 Modbus TCP / OPC UA 客户端。

  3. 无配置外部化

  pos.py、config.py 硬编码。工业部署需 YAML/JSON 配置文件，支持现场调整传感器位置、皮带长度等参数而不改代码。

  🟡 重要级

  4. 无生命周期管理

  缺少：启动自检 → 初始化 → 运行 → 暂停/恢复 → 安全停机 的状态机。直接 ctrl+C 退出不可接受。

  5. 无结构化日志

  print() 散落各处。需分级日志（DEBUG/INFO/WARN/ERROR）+ 文件轮转 + 远程采集。

  6. 无双模式切换

  仿真/真实PLC之间切换需改代码。应支持配置项一行切换：

  mode: simulation  # 或 modbus_tcp / opc_ua

  7. 无 watchdog / 心跳

  PLC断连后无检测、无自动重连、无安全回退。

  🟢 建议级

  8. 无单元测试
  9. 无用户权限管理
  10. 无数据记录/历史趋势
